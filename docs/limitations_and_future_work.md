# Limitations and Future Work

## Current limitations

- Runtime adapter parity is not identical across all platforms.
- Some power/thermal probes depend on hardware-specific tooling.
- Sample outputs are illustrative and not exhaustive of all device/runtime combinations.

## Future work

- Expand native adapter coverage with stricter plugin contracts.
- Add CI automation for non-hardware-dependent validation paths.
- Introduce richer visualization assets and dashboard-style reports.
- Add additional model families and quantization pathways.
